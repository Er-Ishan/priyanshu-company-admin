1. Introduction

The FleetCart API enables partners to retrieve available airports and parking products with calculated prices in real time.

EndpointURLPurpose
Get Airports /api/getAirports List airports for your company
Get Products /api/products Fetch products and live pricing
Security note: Always call the FleetCart API from your server, never from browser JavaScript. Tokens must remain private.

2. Credentials Setup

Before making any API call, obtain both tokens below.

From FleetCart — Service Provider

app_token

Identifies your parking company. Required for every API request.

How to obtain

  1. Contact FleetCart support or your account manager
  2. Request API access for your company
  3. FleetCart activates your account and shares your unique token
  4. Store it securely in environment variables — never in frontend code
From Company Admin Panel

supplier_token

Identifies the supplier/partner. Required for the Products endpoint only.

How to generate

  1. Log in to the FleetCart Company Admin Panel
  2. Go to Suppliers → create or edit a supplier
  3. Ensure the supplier is Active
  4. Copy the generated supplier_token
  5. Configure commission and allowed products as needed

3. Integration Overview

YOUR WEBSITE / APP │ ├─── (1) POST /api/getAirports { app_token } │ └── Returns: airport list (name, IATA, id) │ └─── (2) POST /api/products { app_token, supplier_token, dates, airport } └── Returns: products with prices & commission

4. Quick Start — 5 Steps

  1. Obtain your app_token from FleetCart
  2. Generate a supplier_token in the Company Admin Panel
  3. Call Get Airports to populate your airport dropdown
  4. Collect drop-off/return date & time from the user
  5. Call Get Products and display names and prices

5. Get Airports API

POST https://apis.fleetcart.co.uk/api/getAirports

Request Body

FieldTypeRequiredDescription
app_tokenstringYesCompany token from FleetCart

Example Request

{
  "app_token": "your_app_token_here"
}

Success Response 200 OK

{
  "success": true,
  "data": [
    {
      "airport_id": 1,
      "airport_name": "London Heathrow",
      "iata_code": "LHR"
    },
    {
      "airport_id": 2,
      "airport_name": "London Gatwick",
      "iata_code": "LGW"
    }
  ]
}
When calling /api/products, use the exact airport_name value — not the IATA code alone.

6. Get Products & Pricing API

POST https://apis.fleetcart.co.uk/api/products

Request Body

FieldTypeRequiredDescription
app_tokenstringYesCompany token from FleetCart
supplier_tokenstringYesSupplier token from Admin Panel
depart_datestringYesDrop-off date — YYYY-MM-DD
depart_timestringYesDrop-off time — HH:mm or HH:mm:ss
return_datestringYesReturn date — YYYY-MM-DD
return_timestringYesReturn time — HH:mm or HH:mm:ss
airportstringYesAirport name from Get Airports response

Example Request

{
  "app_token": "your_app_token_here",
  "supplier_token": "your_supplier_token_here",
  "depart_date": "2026-06-22",
  "depart_time": "10:00",
  "return_date": "2026-06-29",
  "return_time": "18:00",
  "airport": "London Heathrow"
}

Success Response 200 OK

{
  "success": true,
  "data": [
    {
      "CompanyName": "Example Parking Ltd",
      "ProductName": "Meet & Greet Terminal 5",
      "ProductPrice": "89.50",
      "AvailableSlots": "",
      "SupplierCommission": 12.5
    }
  ]
}

Response Fields

FieldDescription
CompanyNameYour company name
ProductNameParking product display name
ProductPriceCalculated price (2 decimal places, string)
AvailableSlotsReserved — currently empty
SupplierCommissionCommission % configured for the supplier

7. Step-by-Step Implementation

Step 1 — Store credentials securely

# .env (server-side only — never commit to Git)
FLEETCART_APP_TOKEN=your_app_token_here
FLEETCART_SUPPLIER_TOKEN=your_supplier_token_here
FLEETCART_API_BASE=https://apis.fleetcart.co.uk/api

Step 2 — Load airports on page init

Call Get Airports from your backend and populate a dropdown using airport_name as the value.

Step 3 — Build the search form

UI FieldAPI ParameterExample
Drop-off datedepart_date2026-06-22
Drop-off timedepart_time10:00
Return datereturn_date2026-06-29
Return timereturn_time18:00
AirportairportLondon Heathrow

Step 4 — Fetch products on search

When the user submits the form, your backend calls /api/products and returns results to the frontend.

Step 5 — Display results

Show ProductName and ProductPrice to users. Use SupplierCommission internally for reporting.

An empty data: [] array is a valid response — show a friendly "no availability" message rather than an error.

8. Code Examples

cURL — Get Airports

curl -X POST "https://apis.fleetcart.co.uk/api/getAirports" \
  -H "Content-Type: application/json" \
  -d '{"app_token": "YOUR_APP_TOKEN"}'

cURL — Get Products

curl -X POST "https://apis.fleetcart.co.uk/api/products" \
  -H "Content-Type: application/json" \
  -d '{
    "app_token": "YOUR_APP_TOKEN",
    "supplier_token": "YOUR_SUPPLIER_TOKEN",
    "depart_date": "2026-06-22",
    "depart_time": "10:00",
    "return_date": "2026-06-29",
    "return_time": "18:00",
    "airport": "London Heathrow"
  }'

Node.js — Server-side proxy

app.post("/api/parking-quotes", async (req, res) => {
  const { depart_date, depart_time, return_date, return_time, airport } = req.body;

  const response = await fetch("https://apis.fleetcart.co.uk/api/products", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      app_token: process.env.FLEETCART_APP_TOKEN,
      supplier_token: process.env.FLEETCART_SUPPLIER_TOKEN,
      depart_date, depart_time, return_date, return_time, airport,
    }),
  });

  const result = await response.json();
  res.status(response.status).json(result);
});

PHP — Get Airports

$ch = curl_init('https://apis.fleetcart.co.uk/api/getAirports');
curl_setopt_array($ch, [
    CURLOPT_POST => true,
    CURLOPT_HTTPHEADER => ['Content-Type: application/json'],
    CURLOPT_POSTFIELDS => json_encode(['app_token' => getenv('FLEETCART_APP_TOKEN')]),
    CURLOPT_RETURNTRANSFER => true,
]);
$result = json_decode(curl_exec($ch), true);
curl_close($ch);

9. Error Handling

All errors return JSON: { "success": false, "message": "..." }

StatusMeaningCommon Cause
200SuccessIncludes empty product lists
400Bad RequestMissing fields or invalid date/time
401UnauthorizedInvalid or missing tokens
500Server ErrorRetry or contact FleetCart support

Common Error Messages

MessageAction
app_token is requiredInclude app_token in request body
Invalid app_tokenVerify token with FleetCart; check company is active
supplier_token is requiredInclude supplier_token for /api/products
Invalid supplier_tokenCheck Admin Panel — supplier must be active
Invalid depart or return date/timeUse YYYY-MM-DD and HH:mm formats

10. FAQ & Support

Can I use GET instead of POST?

No. Both endpoints require POST with a JSON body.

Should I send IATA code or airport name?

Send the full airport_name exactly as returned by Get Airports.

Why are some products missing?

Products are filtered by operational hours, airport, supplier settings, and price band availability.

Health check

GET https://apis.fleetcart.co.uk/health

Contact

For API access, token issues, or integration support, contact your FleetCart account manager or support team. When reporting issues, include the endpoint, HTTP status, error message, and timestamp — but never share your tokens.
